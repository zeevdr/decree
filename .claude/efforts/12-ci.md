## Workflows

### 1. PR Validation (`on: pull_request`)

Runs on every PR to main:

| Step | Tool | Purpose |
|------|------|---------|
| Lint (Go) | `golangci/golangci-lint-action` | Code quality |
| Lint (Proto) | `buf lint && buf breaking` (Docker tools image) | Proto compatibility |
| Unit tests | `go test ./internal/... -race` | Correctness |
| SDK tests | `go test ./sdk/...` via workspace | SDK correctness |
| CLI tests | `cd cmd/ccs && go test ./...` | CLI correctness |
| Build server | `go build ./cmd/server` | Compile check |
| Build CLI | `cd cmd/ccs && go build .` | Compile check |
| Docs check | `make docs && git diff --exit-code docs/` | Generated docs up-to-date |
| E2E tests | `make e2e` | Full stack integration |

### 2. Release (`on: push tags v*`)

Runs on version tags:

| Step | Purpose |
|------|---------|
| All PR validation steps | Ensure release is clean |
| Build + push server image | `ghcr.io/zeevdr/central-config-service:<tag>` + `latest` |
| Build + push CLI image | `ghcr.io/zeevdr/ccs:<tag>` + `latest` |

### 3. Main push (`on: push to main`)

| Step | Purpose |
|------|---------|
| Build + push server image | `ghcr.io/zeevdr/central-config-service:main` (bleeding edge) |
| Build + push CLI image | `ghcr.io/zeevdr/ccs:main` |
| Deploy docs | MkDocs â†’ GitHub Pages (if docs/ changed) |

## Considerations

- **Go workspace**: `actions/setup-go` handles `go.work` natively
- **Docker layer caching**: use `docker/build-push-action` with GitHub Actions cache
- **buf breaking**: use `actions/checkout` with `fetch-depth: 0` for full git history
- **E2E timeout**: 10 min timeout for docker compose stack
- **Private repo**: GitHub Actions minutes limited to 2000/month (plenty for now)
